# HTTP 쿠키 (Cookie)
> 쿠키란? 
- 클라이언트(브라우저)에 저장되는 작은 데이터 조각입니다.
- 서버가 클라이언트에게 전송하며, 이후 클라이언트가 서버에 요청할 때 자동으로 포함하여 전송됩니다.

> 쿠키의 주요 특징
- 저장 위치 : 클라이언트(브라우저)
- 저장 형태 : 키-값 쌍 (예: sessionId=abc123)
- 목적 : 로그인 상태 유지, 사용자 정보 저장, 트래킹 등
- 전송 대상 : 서버
- 수명 : 만료 시간 설정 가능 (세션 쿠키 : 브라우저 종료 시 삭제)
- 보안 문제 : 민감 정보 저장 시 보안 위험 (HTTPS 사용 권장)

> 쿠키 사용 예시
1. 로그인 유지
   - 사용자가 로그인할 때, 서버가 세션 ID를 쿠키에 저장하여 브라우저로 전송합니다.
   - 이후 요청 시 쿠키에 포함된 세션 ID로 사용자를 식별합니다.
2. 사용자 맞춤 정보 제공
    - 다크 모드 여부, 언어 설정 등 개인화 설정을 쿠키로 저장하여 사용자가 동일 환경에서 접속하도록 지원합니다.

# HTTP 캐시 (Cache)
> 캐시란?
- 웹 페이지, 이미지, CSS 등 리소스를 클라이언트나 프록시 서버에 저장하여 서버 요청 없이 빠르게 로드할 수 있도록 하는 기법입니다.

> 캐시의 주요 특징
- 저장 위치 : 클라이언트(브라우저), 프록시 서버
- 저장 형태 : 웹 리소스 (HTML, CSS, JS 등)
- 목적 : 서버 부하 감소, 네트워크 트래픽 절약, 빠른 페이지 로딩
- 전송 대상 : 클라이언트 또는 프록시 서버 (요청 생략 가능)
- 수명 : 캐시 만료 시간 또는 조건 설정
- 보안 문제 : 민감한 정보 캐시 시 보안 위험 (캐시 방지 설정 필요)

> 캐시 사용 예시
1. 정적 리소스 캐싱
   - 이미지, CSS 파일 등을 브라우저에 캐시하여 재방문 시 로딩 속도 향상.
2. API 응답 캐싱
    - 변하지 않는 데이터(API 응답)를 클라이언트에 캐시하여 반복 요청을 줄임.
    - 예: 공지사항 리스트, 인기 게시물 등

# 쿠키와 캐시의 사용 사례
## 쿠키가 적합한 경우
- 로그인 상태 유지 : 로그인 후 세션 ID를 저장하여 자동 로그인 구현
- 사용자 환경 설정 : 언어, 테마 등 개인화 정보 저장
- 트래킹 : 광고 분석 도구에서 사용자 행동 데이터 수집

## 캐시가 적합한 경우
- 정적 파일 로딩 속도 개선 : 이미지, CSS, JS 등 자주 사용하는 파일 캐시
- API 응답 캐싱 : 자주 변경되지 않는 데이터를 서버 부하 감소 목적으로 캐시
- 프록시 서버 캐싱 : 여러 사용자에게 동일한 콘텐츠를 제공할 때 중간 서버에서 캐시

# 정리 : 쿠키와 캐시의 역할 구분
## 쿠키
- 주로 사용자 상태 관리에 사용
- 로그인 정보, 사용자 설정 등을 저장하여 반복적인 인증 절차를 줄입

## 캐시
- 주로 데이터 재사용에 사용
- 웹 리소스를 미리 저장하여 로딩 속도 개선 및 트래픽 절감

- 쿠키는 상태 유지용, 캐시는 성능 개선용
